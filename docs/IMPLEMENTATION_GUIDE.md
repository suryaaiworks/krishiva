* Clean architecture
* Responsive UI

# UI Rules

Never redesign completed pages.

Maintain consistent spacing.

Maintain design language.

Use the existing design system.

Follow Material Design principles.

---

# Component Rules

Before creating a component:

Check whether it already exists.

If it exists,

Reuse it.

Never duplicate components.

---

# File Creation Rules

Never generate unnecessary files.

Never generate placeholder files.

Create only files required for the requested task.

---

# Feature Rules

Implement one feature at a time.

Complete it before starting another.

Never leave partially implemented features.

---

# Output Format

Always provide:

1. Files created
2. Files modified
3. Explanation
4. Testing instructions
5. Next recommended task
